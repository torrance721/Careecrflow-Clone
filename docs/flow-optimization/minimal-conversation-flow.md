# 极简对话式开场流程设计

**文档日期**：2026年1月5日  
**版本**：4.0  
**设计理念**：Less structure，更自然的对话

---

## 一、设计理念

### 从"选择"到"对话"

之前的方案都需要用户从预设选项中选择，这有几个问题：

1. 选项可能不完全匹配用户的情况
2. 用户需要"归类"自己，可能感觉被标签化
3. 无法捕捉用户的具体细节和紧急程度

**新方案**：让用户用自己的话说出情况，AI 来理解和适配。

### 核心原则

1. **极简**：只问两个问题
2. **开放**：让用户自由表达
3. **一轮**：不追问，用户说多少算多少
4. **智能**：AI 根据 context 生成针对性内容

### 用户输入示例

**Emily（应届生）可能会写**：
> "我是今年毕业的CS学生，投了很多前端和全栈的岗位，已经收到两个面试邀请了。我比较担心技术问题答不好，还有项目经历讲不清楚。"

**Marcus（转行者）可能会写**：
> "我之前做营销的，现在想转数据分析。投了十几家公司，有几个回复了。最担心的是怎么回答为什么转行，还有技术问题可能答不上来。"

**Priya（国际学生）可能会写**：
> "I'm an international student from India, looking for BA or PM roles. Applied to many companies but not many responses, probably because of visa. Worried about how to answer visa questions and my English expression."

**Alex（训练营）可能会写**：
> "我是训练营毕业的，没有CS学位。投了很多初级前端岗位，回复率很低。最担心技术基础问题，还有怎么解释我的背景。"

### AI 如何处理

AI 会从用户描述中提取关键信息：

| 用户 | 提取的信息 | 生成的面试重点 |
|------|------------|----------------|
| Emily | 应届CS、前端/全栈、担心技术和项目 | 技术基础 + 项目经历讲解 |
| Marcus | 转行、营销→数据、担心转行问题 | 转行相关问题 + 数据分析基础 |
| Priya | 国际学生、BA/PM、签证担忧、英语表达 | 签证问题 + 商业分析 + 表达练习 |
| Alex | 训练营、无学位、前端、技术基础弱 | 技术基础 + 如何展示非传统背景 |

## 三、引导提示设计

Q2 的引导提示非常重要，需要让用户知道可以说什么，但又不限制他们。

### 主提示

> "说说你现在的情况"

### 引导提示（灰色小字）

> "比如你在找什么工作、投了哪些公司、有什么担心的..."

### 备选引导提示

如果需要更详细的引导，可以用：

> "你可以说说：
> - 你在找什么类型的工作
> - 目前投了哪些公司，进展如何
> - 面试中最担心什么问题"

但这个可能太结构化了，建议先用简单版本。

## 四、边界情况处理

### 用户什么都不写

如果用户直接点"开始面试练习"而不写任何内容：

**处理方式**：生成通用面试练习，覆盖最常见的问题类型。

### 用户写得很简单

比如用户只写："找工作"

**处理方式**：AI 根据有限信息生成通用面试，但在面试过程中可以通过问题了解更多。

### 用户写得很详细

比如用户写了一大段：

**处理方式**：AI 充分利用所有信息，生成高度针对性的面试内容。

### 有简历 vs 没简历

| 情况 | Context | 面试针对性 |
|------|---------|------------|
| 有简历 + 详细描述 | 最丰富 | 最高 |
| 有简历 + 简单描述 | 较丰富 | 较高 |
| 没简历 + 详细描述 | 中等 | 中等 |
| 没简历 + 简单描述 | 最少 | 通用 |

## 五、与之前方案的对比

| 维度 | 方案 C+E（选择式） | 新方案（对话式） |
|------|-------------------|------------------|
| 开场步骤 | 从多个选项中选择 | 两个问题 |
| 用户表达 | 受限于预设选项 | 自由表达 |
| 信息丰富度 | 固定 | 取决于用户 |
| 自然程度 | 像填表 | 像聊天 |
| AI 工作量 | 低（规则匹配） | 高（理解+生成） |
| 个性化程度 | 中等 | 高 |

## 六、技术实现要点

### Prompt 设计

```
你是 UHired 面试教练。根据以下用户信息，生成针对性的面试练习。

## 用户简历（如有）
{resume_content}

## 用户描述
{user_description}

## 你的任务
1. 分析用户的背景、目标职位、紧急程度、关注点
2. 生成 5-8 个针对性的面试问题
3. 问题应该覆盖三个层次：答对、答好、应对
4. 如果用户有特殊情况（转行/国际学生/训练营），包含相关问题

## 输出格式
...
```

### 简历解析

如果用户上传简历，需要：
1. 提取关键信息（教育背景、工作经历、技能）
2. 作为 context 的一部分传给 AI
