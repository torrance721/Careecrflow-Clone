## 执行摘要

经过多轮流程设计和四位 Persona 的测试评价，我们确定了 UHired 的最终用户流程：**Job-First Flow（职位先行）**。

核心理念是：**用户来 UHired 是为了找工作，不是做职业规划**。因此，流程应该直接展示职位，而不是先收集用户信息。

|------|--------|--------|
| Persona 平均分 | 4.08/5 | **4.7/5** |
| 看到职位的时间 | 3-5分钟 | **10秒** |
| 第一步 | 选择用户状态 | 搜索职位 |
| 核心问题 | "你是什么状态？" | "你想找什么工作？" |

## 最终流程

### 流程概览

```
搜索职位 → 查看职位列表 → 上传简历（可选）→ 匹配度分析 → 针对性面试 → 评估报告 → 行动建议
```

### 详细步骤

**第一步：搜索职位（10秒）**

用户进入 UHired 后，第一个界面是职位搜索框：
- 输入职位关键词（如"Frontend Developer"）
- 选择工作地点（如"San Francisco"）
- 点击搜索

**第二步：查看职位列表（立刻展示价值）**

展示从 LinkedIn 抓取的真实职位：
- 公司名称和 Logo
- 职位名称
- 薪资范围
- 发布时间
- 工作类型（全职/远程等）

**第三步：上传简历（可选但推荐）**

用户可以选择：
- 上传简历（PDF/Word）
- 从 LinkedIn 导入
- 快速填写（2分钟）
- 通过对话生成简历

不上传简历也能继续，但会提示"上传简历可以看到匹配度和个性化建议"。

**第四步：匹配度分析**

基于简历和职位 JD，展示：
- 整体匹配度（百分比）
- 符合要求的点（绿色）
- 需要准备的点（黄色）
- 差距（红色）

**第五步：针对性面试**

根据职位要求和用户差距，定制面试内容：
- 行为面试（Behavioral）
- 技术面试（Technical）
- 系统设计（如适用）

面试问题针对具体职位，而不是泛泛的"前端面试"。

**第六步：评估报告 + 行动建议**

面试结束后展示：
- 各环节得分
- 具体改进建议
- 推荐学习资源
- 预计准备时间

## 没有简历的用户

对于没有简历的用户，提供两个路径：

### 路径 A：快速填写（2分钟）

通过 4-5 个选择题快速收集信息：
1. 最高学历和专业
2. 工作/实习经验年限
3. 技能列表（多选）
4. 项目经验（简述）

### 路径 B：对话生成简历

通过 AI 对话收集信息并生成简历：
- 3-5 轮对话
- AI 提问用户的经历、项目、技能
- 最后生成简历草稿供用户确认
