# UHired 用户流程重设计：以找工作为核心

**文档日期**：2026年1月5日  
**核心理念**：职位先行，简历为本，面试为用  
**状态**：待评审

---

## 问题诊断

### 之前流程的问题

之前的流程设计偏向"职业规划"思维：
- 问用户"你是什么状态"
- 问用户"你对什么方向感兴趣"
- 问用户"你的职业目标是什么"

这些问题太抽象，不是用户进来的真正诉求。

### 用户的真正诉求

用户打开 UHired 的心态是：**"我想找到一份工作"**

他们想要的是：
1. 看到适合我的**真实职位**
2. 准备好**面试**拿到 offer
3. **越快越好**

## 关键设计决策

### 1. 职位搜索作为入口

用户进来第一件事就是搜索职位，而不是回答问题。这样：
- 立刻展示产品价值（真实职位）
- 符合用户"找工作"的心理预期
- 降低使用门槛

### 2. 简历是可选但推荐的

- 不上传简历也能看职位和开始面试
- 上传简历后解锁：匹配度分析、个性化建议、针对性面试
- 这样既不阻挡用户，又激励用户上传简历

### 3. 面试针对具体职位

- 不是泛泛的"前端面试"
- 而是"Google Frontend Engineer L3 面试"
- 根据 JD 要求和用户简历差距，定制面试内容

### 4. 评估报告导向行动

- 不只是给分数
- 明确告诉用户"下一步做什么"
- 给出具体的学习资源和时间预估

## 没有简历的用户流程

对于没有简历的用户，提供两个选择：

### 选择 A：快速填写（2分钟）

```
Q1: 你的最高学历是？
    [本科 ▼] [计算机科学 ▼]

Q2: 你有工作/实习经验吗？
    ○ 有，[1] 年
    ○ 没有，但有项目经验
    ○ 都没有

Q3: 你会哪些技能？
    ☑ JavaScript  ☑ React  ☐ Python  ☐ Java  [+ 添加]

[生成简历摘要]
```

### 选择 B：对话生成简历

```
AI: "让我帮你整理一份简历。先告诉我，你最近做过什么项目或工作？"

用户: "我在学校做了一个电商网站项目，用 React 和 Node.js..."

AI: "听起来不错！这个项目你主要负责什么？遇到过什么挑战？"

...（3-5轮对话）

AI: "根据我们的对话，我帮你生成了一份简历草稿，你看看需要修改吗？"
```
