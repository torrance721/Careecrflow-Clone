**文档日期**：2026年1月5日  
**核心理念**：先评估，再推荐，再练习；帮用户建立正确的面试认知  
**状态**：待评审

---

## 用户画像

我们的目标用户具有以下特征：

**基本特征**：
- 年轻（应届生/初入职场 1-3 年）
- 想进好公司（大厂、知名企业）
- 面试能力不足，缺乏信心
- 不确定该投哪些职位
- 担心面试表现不好

**心理特征**：
- 把面试当成**考试**，害怕"考砸"
- 认为面试问题都是在"考核"自己
- 追求"标准答案"，担心说错
- 紧张、焦虑，缺乏自信

**真实需求**：
- 想知道自己现在的水平够不够
- 想知道该投什么级别的职位
- 想在安全的环境里练习面试
- 想建立信心，不再害怕面试

## 核心洞察

### 洞察一：面试不是考试

年轻人最大的误解是把面试当成考试。但实际上：

| 考试 | 面试 |
|------|------|
| 有标准答案 | 没有标准答案 |
| 考官想难倒你 | 面试官想了解你 |
| 答错扣分 | 真实表达加分 |
| 单向考核 | 双向沟通 |
| 背诵知识 | 展示自己 |

### 洞察二：很多面试问题是为了获取信息

| 面试官问 | 用户以为 | 实际目的 |
|----------|----------|----------|
| “介绍一下你自己” | 考我表达能力 | 快速了解你的背景和亮点 |
| “你的缺点是什么” | 挖坑让我跳 | 看你有没有自我认知和成长意识 |
| “为什么想来我们公司” | 考我做没做功课 | 了解你的求职动机和匹配度 |
| “说说你遇到的困难” | 考我解决问题能力 | 了解你的工作风格和思维方式 |
| “你有什么问题问我” | 考我准备充分吗 | 真的想回答你的疑问，看你关心什么 |

### 洞察三：面试问题的三个层次

面试问题不只是“答对”就够了，还有“答好”的空间：

**第一层：客观题（答对）**

有明确答案，需要答对：
- “React 的虚拟 DOM 是什么？”
- “HTTP 状态码 404 是什么意思？”
- “SQL 的 JOIN 有哪几种？”

目标：答对 ✓

**第二层：客观题 + 主观表达（答好）**

有客观答案，但可以答得更好。以 “React 的虚拟 DOM 是什么？” 为例：

| 层次 | 回答示例 |
|------|----------|
| 答对 | “虚拟 DOM 是内存中的 DOM 树表示，用于减少真实 DOM 操作” |
| 答好 | “虚拟 DOM 是... 在我之前的项目里，我们用它优化了列表渲染，性能提升了 40%” |
| 答得更好 | “虚拟 DOM 是... 不过在某些场景下，比如大量动态节点，我发现直接操作 DOM 反而更快，所以我会根据场景选择...” |

目标：
- 答对 ✓
- 展示实战经验 ✓
- 体现思考深度 ✓
- 展示自己的优势 ✓

**第三层：主观题（应对）**

没有对错，看你怎么思考和表达：
- “你怎么看待微服务架构？”
- “如果让你重新设计这个系统，你会怎么做？”
- “你和同事意见不一致时怎么处理？”

目标：
- 有清晰的思路 ✓
- 能说出理由 ✓
- 展示自己的风格 ✓

### 核心洞察

**很多人只追求“答对”，但面试官更看重的是“答好”**——你的思考深度、实战经验、解决问题的方式。

UHired 的价值是帮用户从“答对”升级到“答好”。

### 洞察四：用户需要的是信心

用户来 UHired 不只是想"提升面试技巧"，更深层的需求是：
- 想知道"我够不够格"
- 想在安全的地方"试错"
- 想听到"你准备好了"

## 流程设计

### 流程概览

```
欢迎 + 建立信任
      ↓
快速了解背景（2-3个问题）
      ↓
摸底面试（10分钟）
      ↓
能力评估 + 职位定位
      ↓
推荐适合的职位方向
      ↓
针对性练习
      ↓
追踪进步 + 建立信心
```

### 第一步：欢迎 + 建立信任

用户进入 UHired 后，第一个界面不是让他填表或选择，而是一段温暖的欢迎语：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                          │
│  👋 你好！欢迎来到 UHired                                               │
│                                                                          │
│  我是你的 AI 面试教练。                                                 │
│                                                                          │
│  很多人把面试当成考试，害怕答错、担心表现不好。                         │
│  但其实，面试只是一次对话——面试官想了解你，你也在了解公司。             │
│                                                                          │
│  在这里，你可以：                                                        │
│  ✓ 安全地练习，答错没关系                                               │
│  ✓ 了解自己的真实水平                                                   │
│  ✓ 知道该投什么级别的职位                                               │
│  ✓ 每次练习都会进步                                                     │
│                                                                          │
│  准备好了吗？让我们开始吧。                                             │
│                                                                          │
│  [开始]                                                                  │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

**设计意图**：
- 先建立信任，降低焦虑
- 传递"面试不是考试"的理念
- 强调"安全的练习环境"

│  │  ○ 我工作了 1-3 年，想跳槽到更好的公司                          │    │
│  │  ○ 我在转行，之前做其他领域                                     │    │
│  │  ○ 其他情况                                                      │    │

用户选择后，继续：

**设计意图**：
- 对话式，不是填表
- 允许"不确定"
- 快速收集关键信息

这是核心环节。通过一次模拟面试来评估用户的真实水平。

- 建立信心："你的基础不错"

## 核心差异化

### 与其他面试产品的区别

| 维度 | 其他产品 | UHired |
|------|----------|--------|
| 定位 | 面试题库/刷题 | 面试教练 |
| 方式 | 背答案 | 理解 + 练习 |
| 反馈 | 对/错 | 为什么 + 怎么改 |
| 心态 | 考试 | 对话 |
| 目标 | 答对题 | 建立信心 |

### 独特价值

1. **教育用户**：每个问题都解释"面试官为什么问这个"
2. **能力定位**：告诉用户"你现在适合投什么"
3. **建立信心**：可视化进步，给出"准备好了"的信号
4. **安全环境**：强调"答错没关系，这是练习"
