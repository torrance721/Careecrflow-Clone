## 一、用户进入渠道与心态

### 用户如何发现 UHired

UHired 的目标用户通过以下渠道发现产品：

| 渠道 | 用户心态 |
|------|----------|
| 朋友推荐 | "朋友说这个能帮我准备面试" |
| 社交媒体广告 | "这个 app 说能帮我过面试" |
| 求职论坛 | "有人推荐这个面试准备工具" |
| 应用商店搜索 | 搜索"面试准备"、"mock interview" |

### 用户进入时的心态

用户进入 UHired 时的真实心态是：

> "我在找工作，有些大概方向，但也不是非要做什么不可，有份好工作就行。我怕面试不过，所以想找个能帮我过面试的应用。这个叫 UHired 的说能帮我搞好面试，我试试。"

**关键心态特征**：

1. **目标模糊**：有大概方向，但不是非常明确
2. **焦虑面试**：担心面试不过，缺乏信心
3. **实用导向**：想要"过关"，不是想"学习"
4. **尝试心态**：先试试看，不一定会长期使用
5. **多线程求职**：同时面试多家公司

### 用户不会说的话

用户进入时**不会**这样想：

- ❌ "我想做职业规划"
- ❌ "我想了解自己适合什么"
- ❌ "我想系统学习面试技巧"
- ❌ "我想提升自己的综合能力"

## 二、用户的直接诉求 vs 深层需求

### 直接诉求（用户说的）

| 诉求 | 用户原话 |
|------|----------|
| 过面试 | "帮我过面试" |
| 准备答案 | "告诉我怎么回答这些问题" |
| 练习机会 | "让我练练面试" |
| 快速见效 | "我下周就要面试了" |

### 深层需求（用户没说的）

| 需求 | 实际含义 |
|------|----------|
| 信心 | "我想知道自己够不够格" |
| 方向 | "我不确定该投什么职位" |
| 安全感 | "我想在安全的地方试错" |
| 认可 | "我想有人告诉我'你准备好了'" |

### UHired 的价值定位

| 维度 | 表面价值（用户期待） | 实际价值（产品提供） |
|------|----------------------|----------------------|
| 功能 | 帮我过面试 | 提升真实面试能力 |
| 内容 | 给我面试技巧 | 建立正确的面试认知 |
| 结果 | 让我答对题 | 让你从"答对"到"答好" |
| 终极 | 帮我找到好工作 | 让你理解什么工作对你最好，并帮你找到它 |

**产品策略**：用"帮你过面试"吸引用户，通过练习和反馈真正提升能力和信心。

## 三、Persona 核心特征

### 必须具备的特征

每个 Persona 都必须具备以下特征：

1. **年轻**：22-28岁，职场新人或准新人
2. **想进好公司**：有上进心，目标是知名企业
3. **面试焦虑**：担心面试表现不好
4. **目标模糊**：有大概方向，但不是非常明确
5. **愿意尝试**：愿意用新工具帮助自己

### 差异化维度

不同 Persona 在以下维度有所不同：

| 维度 | 可能的变化 |
|------|------------|
| 背景 | 应届生、在职跳槽、转行、国际学生 |
| 经验 | 无经验、1-2年、实习经验 |
| 技能 | 技术、商业、设计、运营 |
| 焦虑程度 | 轻度、中度、重度 |
| 时间紧迫度 | 不急、一般、很急 |

## 四、Persona 完整模板

生成 Persona 时，必须包含以下所有部分：

### 4.1 基本信息

```markdown
## 基本信息

| 项目 | 内容 |
|------|------|
| 姓名 | [中文名（英文名）] |
| 年龄 | [22-28岁] |
| 当前状态 | [在校/应届/在职/失业/转行] |
| 教育背景 | [学校、专业、学历] |
| 工作经验 | [年限、类型] |
```

### 4.2 求职情况

```markdown
## 求职情况

**大概方向**：[职位方向，不需要非常明确]

**目标公司**：[大厂/中型/创业公司，具体公司名]

**时间紧迫度**：[不急/一般/比较急/很急]

**已有准备**：[已做的准备工作]
```

### 4.3 心态描述

```markdown
## 心态描述

### 进入 UHired 的原因
[为什么下载/打开这个 app]

### 直接期待
> "[用户原话，希望 UHired 帮他做什么]"

### 内心焦虑
[担心什么，列出具体焦虑点]

### 隐藏需求
[没说出口但真正需要的]
```

### 4.4 典型内心独白

```markdown
## 典型内心独白

> "[用第一人称描述用户的真实想法，2-3段]"
```

### 4.5 使用场景

```markdown
## 使用场景

**场景**：[具体使用场景描述]

**期待**：
1. [期待1]
2. [期待2]
3. [期待3]

**可能的行为**：
- [行为1]
- [行为2]
- [行为3]
```

### 4.6 评价倾向

```markdown
## 评价倾向

[Persona 名] 会关注：

1. **[维度1]**：[具体关注点]
2. **[维度2]**：[具体关注点]
3. **[维度3]**：[具体关注点]
4. **[维度4]**：[具体关注点]
```

## 五、简历生成模板（必须）

**每个 Persona 必须附带一份简历**，用于测试简历上传和解析功能。

### 5.1 简历文件命名规则

```
docs/personas/resumes/[persona-id]-[name]-resume.md
docs/personas/resumes/[persona-id]-[name]-resume.pdf
```

例如：
- `docs/personas/resumes/05-xiaowang-resume.md`
- `docs/personas/resumes/05-xiaowang-resume.pdf`

### 5.2 简历模板

```markdown
# [姓名]

**Email:** [email] | **Phone:** [phone] | **Location:** [location]

---

## 教育背景

**[学校名称]** | [时间]  
[专业] [学历]  
GPA: [GPA]

**相关课程：** [课程列表]

---

## 技能

- **[技能类别1]：** [具体技能]
- **[技能类别2]：** [具体技能]
- **[技能类别3]：** [具体技能]

---

## 实习/工作经历

### [公司名] - [职位] | [时间]

- [工作内容1]
- [工作内容2]
- [工作内容3]

### [公司名] - [职位] | [时间]

- [工作内容2]

## 项目经历

### [项目名]（[项目类型]） | [时间]
```

### 5.3 简历内容要求

简历内容必须与 Persona 描述一致：

| Persona 描述 | 简历体现 |
|--------------|----------|
| 教育背景 | 学校、专业、GPA |
| 工作经验 | 实习/工作经历 |
| 技能水平 | 技能列表 |
| 项目经历 | 项目描述 |
| 已有准备 | 证书、课程 |

### 5.4 简历生成步骤

1. 根据 Persona 基本信息填写简历模板
2. 保存为 Markdown 文件
3. 使用 `manus-md-to-pdf` 转换为 PDF
4. 两个文件都保存在 `docs/personas/resumes/` 目录

## 六、评价标准

### Persona 测试流程时的评价维度

1. **第一印象**：进入 app 的第一感觉是否符合预期
2. **价值感知**：是否觉得这个 app 能帮到自己
3. **焦虑缓解**：使用后是否感觉更有信心
4. **继续使用意愿**：是否愿意继续用、推荐给朋友
5. **具体反馈**：哪里好、哪里需要改进

### 评分标准

| 分数 | 含义 |
|------|------|
| 5 | 完全符合预期，会持续使用并推荐 |
| 4 | 基本满意，会继续尝试 |
| 3 | 一般，可能会用也可能不用 |
| 2 | 不太满意，可能会卸载 |
| 1 | 完全不符合预期，立即卸载 |

## 七、注意事项

### 避免的错误

1. **不要让 Persona 太理性**：真实用户是焦虑的、急躁的
2. **不要让 Persona 目标太明确**：真实用户是模糊的
3. **不要让 Persona 太有耐心**：真实用户想快速见效
4. **不要让 Persona 太配合**：真实用户会质疑、会跳过

### 保持真实

Persona 的反馈应该反映真实用户的心态：

- 会跳过不想看的内容
- 会质疑"这有用吗"
- 会在焦虑时做出非理性选择
- 会因为一点小问题而放弃

生成新 Persona 时，确保完成以下所有项目：

- [ ] 基本信息完整
- [ ] 求职情况描述清晰

- [ ] 典型内心独白真实可信
- [ ] 使用场景具体
- [ ] 评价倾向明确
- [ ] **简历 Markdown 文件已创建**
- [ ] **简历 PDF 文件已生成**
- [ ] 简历内容与 Persona 描述一致
